Future

L’interfaccia Future<V> permette di rappresentare un risultato (di tipo V) che
sarà disponibile in un momento successivo (futuro). Rappresenta un compito che
esegue in modo asincrono ed è un riferimento al risultato
L’avvio di un’attività per mezzo di un Future permette al thread chiamante di
continuare a fare del lavoro, anziché aspettare il risultato dell’operazione

// prima di Java 8
ExecutorService esecutore = Executors.newCachedThreadPool();
Future<Double> risultato = esecutore.submit(new Callable<Double>() {

return calcola();
}

Considerazioni Su Future
•

•

Il thread aggiuntivo è fornito da ExecutorService

•

Quando non si può più procedere senza il risultato dell’operazione che
prende tanto tempo e che sta eseguendo sul thread aggiuntivo, si può
prendere il risultato chiamando get() sul Future (ovvero sulla
variabile risultato)

•

get() restituirà immediatamente il valore se l’operazione era già stata
completata, altrimenti il thread che chiama get() si blocca in attesa

che il risultato diventi disponibile
•

Executors e newCachedThreadPool() permettono di creare un nuovo thread,
ovvero l’istanza di ExecutorService. Tale istanza esegue l’operazione richiesta

(questa operazione, supponiamo, prende tanto tempo per eseguire)
•

Lettura Risultato Da Future: get()

Il metodo get() con un parametro (overloaded) accetta un timeout
che definisce il tempo massimo di attesa per il thread che aspetta il
risultato. Se si raggiunge il tempo massimo di attesa si ha l’eccezione
TimeoutException

•

Il metodo isDone() ritorna un boolean che è true se l’esecuzione è

CompletableFuture allOf()
•

Si vuol aspettare il completamento di tutti i compiti svolti da un certo numero di
esecuzioni asincrone. Primo passo: si avviano le esecuzioni asincrone con il metodo
statico supplyAsync() e si mette il risultato su un elemento di un array di
CompletableFuture

•

Quindi, si usa il metodo statico allOf() passando l’array che ritorna un
CompletableFuture<Void> che è completato quando tutti gli elementi dell’array
hanno completato (allOf() può prendere vari parametri anziché un array)

•

Infine, si può usare join() per aspettare che tutte le esecuzioni abbiano
completato, quest’ultimo ritorna un Void

CompletableFuture<Double>[] prezzi = new CompletableFuture[2];
prezzi[0] = CompletableFuture.supplyAsync(() -> calcolaPrezzo("prodotto0"));
prezzi[1] = CompletableFuture.supplyAsync(() -> calcolaPrezzo("prodotto1"));
CompletableFuture.allOf(prezzi).join();
try {
System.out.println(prezzi[0].get());
System.out.println(prezzi[1].get());
} catch (InterruptedException e) {
e.printStackTrace();
} catch (ExecutionException e) {
e.printStackTrace();

Esecuzione Di Step

thenCombine() è un metodo di CompletableFuture che prende due parametri:
(i) un CompletableFuture fsc, e (ii) una funzione con due argomenti; quindi
restituisce un CompletableFuture

•

Quando il CompletableFuture fpr su cui è chiamato thenCombine() e quello
passato come parametro fsc hanno completato, allora i due risultati sono passati
come argomento alla funzione fornita come secondo parametro; thenCombine()
restituisce un CompletableFuture con il risultato della funzione passata

var fpp = fpr.thenCombine(fsc, (a, b) -> a * b);

var fsc = CompletableFuture.supplyAsync(() -> stepSconto());
var fpr = CompletableFuture.supplyAsync(() -> stepPrezzo());
var fpp = fpr.thenCombine(fsc, (a, b) -> a * b);
•

thenCompose() prende una funzione con un parametro, e restituisce un
CompletableFuture

•

Quando l’esecuzione sul CompletableFuture fpp su cui è chiamato
thenCompose() è completata, chiama la funzione passata e restituisce il
CompletableFuture che è il risultato della funzione chiamata

var fps = fpp.thenCompose(p -> stepCalcola(p));
