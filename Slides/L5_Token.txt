Generazione e Verifica di Token

▸ Una volta autenticato l’utente, gli si fornisce un token che userà per le
sue successive richieste ai servizi

TestToken

▸ Un token per l’accesso è una credenziale che può essere usata per
accedere a servizi
▸ Un token può essere una stringa (opaque, ovvero incomprensibile),
per es. kPoPMRYrCEoYO6s5 oppure un oggetto, per es. un oggetto
JSON, detto JSON Web Token (JWT)
▸ Lo scopo del token è informare che chi lo possiede è stato autorizzato
ad accedere al servizio
▸ Il token tipicamente indica: l’utente identificato, chi lo ha emesso, il
timestamp al momento dell’emissione, la durata, payload, una
signature

JSON Web Token (JWT)

Claim Standard per JWT

▸ I JWT (pronuncia jot) sono un formato standard per token di
sicurezza auto-contenuti

▸ Alcuni claim standard (RFC 7519) sono
▸ Issuer: iss String, chi ha creato il token

▸ Un JWT consiste di un set di diritti (claim) per un utente
rappresentati tramite un oggetto JSON, insieme a un header
(intestazione) che descrive il formato del token

▸ Subject: sub String, l’identificativo univoco dell’utente
▸ Audience: aud String o array, riceventi previsti per il token

▸ I JWT sono protetti crittograficamente e possono essere
criptati

▸ Not before: nbf number, token non valido se usato prima di
questo istante

▸ L’oggetto JSON può contenere gli attributi della sessione;
viene creato dal server, mandato al client, e successivamente
quando il server lo riceve recupera gli attributi della sessione

▸ Issued at: iat number, quando il token è stato creato
▸ Expiration time: exp number, fine della validità del token
▸ Id: jti String, identificativo del token

Verifica del Token

String token = JWT.create()
.withIssuer(“auth0-my-demo")
.withSubject(“my-user-123”)
.withIssuedAt(new Date())
.withExpiresAt(new Date(System.currentTimeMillis() + 3600_000))
.withJWTId("token-xyz-123")
.sign(algorithm);

JWTVerifier verifier = JWT.require(algorithm)
.withIssuer(“auth0-my-demo")
.build();

▸ Il token creato contiene l’id di chi lo ha prodotto, l’id
dell’utente, la data di creazione, la data di scadenza, l’id
del token, e quindi la firma crittografica

▸ Il metodo verify() lancia una eccezione se il token non è
valido

DecodedJWT decoded = verifier.verify(token);
System.out.println("Subject: " + decoded.getSubject());

▸ Il metodo sign() prende una istanza che rappresenta
l’algoritmo di crittografia per la firma e per la verifica
